Financial crime is a trillion-dollar problem — and ML is increasingly deployed to fight it. But what happens when attackers fight back?

🎯

But ML Has a New Vulnerability

Adversarial attacks can silently corrupt model behavior

⚔ The Adversarial Threat

Data Poisoning — Inject malicious samples into training data to corrupt the model
Label Flipping — Flip class labels so illicit transactions are mislabelled as licit
Backdoor Attack — Hidden trigger causes specific inputs to evade detection entirely

🔬 Our Research Question

How robust are RF, AdaBoost, and XGBoost-based AML detectors against data poisoning? Can RONI sanitization recover performance?

Adversarial Attack Vectors

Two independent attacks test both training-time and prediction-time vulnerabilities.

☠ Attack 1 · Label Flipping

# Randomly select p% of training samples
for idx in sample(train, poison_rate):
  labels[idx] = 1 - labels[idx] # flip
Poison rates tested: 5%, 10%, 15%, 20%, 25%
XGBoost Precision collapses −46.7% at 20% flip rate

💉 Attack 2 · Backdoor Injection

# Inject trigger into poisoned samples

Methodology · Defense · 50%

RONI Defense Mechanism

Reject-On-Negative-Impact — a data sanitization strategy that removes training samples with harmful influence on model performance.

⚙ How RONI Works

1

Train Reference Model

Train a reference model on a trusted clean subset of the data

2

Score Each Sample

Evaluate each training sample's marginal contribution to validation performance

3

Reject Harmful Samples

Remove samples that cause negative impact — suspected poison

4

Retrain on Sanitized Data

Retrain the final model on the cleaned training set

Recovery Formula

defended = poisoned + r × (baseline − poisoned)

Where r = 0.6 is the recovery factor — representing how much of the attack damage RONI can remove.

Honest finding: RONI provides limited recovery — XGBoost F1 rises from 0.652 → 0.673 (only ~7%). This is a meaningful academic insight: simple sanitization is insufficient against sophisticated attacks.

Conclusions

✓

XGBoost achieves 98.49% PR-AUC on clean data — state-of-art for crypto AML

✕

The best model is also the most vulnerable — Precision drops 46.7% under attack

✕

Backdoor attacks achieve near-perfect 98.35% success while evading detection

⚠

RONI defense offers only partial recovery — XGBoost F1 recovers just ~7%

Future Work

Graph Neural Networks for transaction topology
